  <action>
    Build the structured analysis display and the markdown export feature.

    **Backend — `backend/app/services/export.py`:**

    Create `async def export_story_markdown(story_id: str) -> str` that:
    1. Loads the story from SQLite (title, premise, active_path)
    2. Parses `active_path` as a JSON array of node IDs
    3. For each node ID in order, loads the node content
    4. Produces a markdown document:
       ```markdown
       # {story.title}

       > **Premise:** {story.premise}

       ---

       {node_1.content}

       {node_2.content}

       {node_3.content}

       ---

       *Exported from AI Invasion on {datetime}*
       ```
    5. Each node's content is separated by a blank line (paragraph breaks)
    6. If active_path is empty or null, export just the premise

    **Backend — Update `backend/app/routers/stories.py`:**

    Add a new endpoint:
    - **GET /api/stories/{story_id}/export** — Returns the markdown export
      - Content-Type: `text/markdown`
      - Content-Disposition: `attachment; filename="{story_title}.md"`
      - Calls `export_story_markdown()` from the export service
      - Returns as a `Response` with proper headers (NOT JSON)

    **Frontend — `frontend/src/lib/components/AnalysisCard.svelte`:**

    Create a reusable card component for a single analysis section:
    - Props: `let { title, children } = $props()` — title is the section header, children is the content snippet
    - Dark themed card: bg-gray-800, rounded-lg, p-3, border border-gray-700
    - Title: text-sm font-semibold text-gray-400 uppercase tracking-wide
    - Content area: text-gray-200, text-sm, leading-relaxed
    - Collapsible: click title to expand/collapse content (collapsed by default for long sections)

    **Frontend — `frontend/src/lib/components/AnalysisPanel.svelte`:**

    Create the structured analysis display for the right sidebar:
    - Reads `generationState.lastAnalysis` to get the latest StoryAnalysis
    - If no analysis available, shows: "Generate a continuation to see story analysis"
    - When analysis is available, renders these cards (using AnalysisCard):

      1. **Logline** — Single line summary. Display as-is.
      2. **Cast** — List of characters. Render as a bulleted list with character names bolded.
      3. **World Rules** — World-building rules. Render as a bulleted list.
      4. **POV & Tone** — Point of view and tense/tone. Display as a short paragraph.
      5. **Timeline** — Story timeline. Render as a numbered or bulleted list of events.
      6. **Current Situation** — What's happening now. Display as a paragraph.
      7. **Active Threads** — Open plot threads. Render as a bulleted list.
      8. **Continuity Landmines** — Potential inconsistencies. Render as a bulleted list with warning styling (amber text).
      9. **Ambiguities** — Unclear elements. Render as a bulleted list.
      10. **Next Paragraph Seeds** — Suggestions for next generation. Render as a numbered list.

    - Add an "Export Markdown" button at the bottom of the panel:
      - Clicks trigger a download of the markdown export: `window.open('/api/stories/${storyState.activeStoryId}/export')`
      - Disabled if no active story
      - Styled: bg-indigo-600 hover:bg-indigo-500 text-white

    - Scrollable container (overflow-y-auto) since analysis can be long
    - All Svelte 5 syntax: `$props()`, `$state()`, `$derived()`, `onclick`

    **Frontend — Update `frontend/src/routes/+layout.svelte`:**
    - Replace the right sidebar placeholder text with the `AnalysisPanel` component
    - Import `AnalysisPanel` and render it inside the right sidebar's content area

  <action>
    Implement story loading into the editor and polish the overall app flow.

    1. **Update `frontend/src/lib/stores/story.svelte.ts`:**
       - Add `async loadStory(id: string)` method:
         - Calls `api.getStory(id)` to fetch the full story with nodes and provenance spans
         - Sets `activeStoryId` and updates the stories array
         - Returns the story data for the editor to consume
       - Add `getActivePathNodes()` derived helper:
         - Reads `activeStory.active_path` (JSON array of node IDs)
         - Returns the nodes in path order for rendering in the editor
       - Add `lastAnalysis = $state<StoryAnalysis | null>(null)` to store the most recent analysis per story (so it persists when switching between stories)
       - On `setActiveStory(id)`, load the story if not already fully loaded (has nodes)

    2. **Update `frontend/src/lib/components/Editor.svelte`:**
       - Add a `loadContent(nodes: StoryNode[], spans: ProvenanceSpan[][])` method:
         - Clears the editor content
         - For each node in order, inserts its content with appropriate provenance marks:
           - For each provenance span, apply the corresponding mark to the character range
           - If a node has no provenance spans, apply the node's `source` field as the default mark for all content
         - Positions cursor at the end
       - Watch for active story changes via `$effect()`:
         - When `storyState.activeStoryId` changes, call `loadContent()` with the active path nodes
         - If activeStoryId becomes null, clear the editor

    3. **Update `frontend/src/routes/+page.svelte`:**
       - On mount, load the story list: `storyState.loadStories()`
       - If there's a previously active story (stored in localStorage), auto-select it
       - Save `activeStoryId` to `localStorage` on change (via `$effect()`) so it persists across page refresh
       - When no story is selected, show a welcome state in the editor area:
         - "Create a new story in the Settings panel to get started"
         - Or a simple "New Story" button that opens the creation form
       - When a story is selected:
         - Editor loads with the story content (from active_path nodes with provenance)
         - GenerationControls enabled
         - Analysis panel shows last analysis (if available)
       - Ensure proper cleanup: disconnect WebSocket and save state on unmount

    IMPORTANT:
    - Story content restoration must preserve provenance marks — each span of text must have the correct color
    - Use `$state.raw()` for node arrays (per research Pitfall 2)
    - User edits to existing text should create `user_edited` provenance spans (track via Tiptap transaction observation)
    - Handle edge cases: empty stories, stories with only a premise, very long stories
