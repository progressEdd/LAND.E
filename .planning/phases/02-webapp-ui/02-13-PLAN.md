<objective>
Fix four visual bugs identified from light/dark mode testing:

![Bug report screenshot](../../../00-supporting-files/images/02-CONTEXT/20260214013425.png)

1. **Story title box black in light mode** — The `.story-item.active` selector uses hardcoded `background-color: #1e1b4b` which appears nearly black on light backgrounds. Replace with CSS custom property using a semi-transparent indigo tint.
2. **Provenance highlights invisible in dark mode** — Background-color tints at 12% opacity are too subtle against dark editor backgrounds. Increase to ~22-25%.
3. **Character name text labels clutter the graph** — Full character names next to supernodes add visual noise. Users can hover for details via tooltips (added in 02-12). Remove the `<text>` labels and reduce `CHAR_COLUMN_X_OFFSET` since no text to accommodate.
4. **Cross-edge lines too faint** — Current 0.18 opacity on `.char-edge` makes connections barely visible. Increase to 0.38.
</objective>
