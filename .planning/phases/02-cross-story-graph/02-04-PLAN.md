# Plan 02-04: Graph Visualization ŌĆö Linked Characters and Enhanced Dashboard

## Objective

Enhance the dashboard graph to render linked (canonical) characters as unified multi-story nodes with visual distinction, add click handlers to open the character profile panel, integrate the match and profile panels into the Dashboard component, and wire up the enhanced overview data flow.

## Tasks

### Task 1: Enhance DashboardGraph.svelte for linked characters

<read_first>
- `02-worktrees/webapp-ui/frontend/src/lib/components/DashboardGraph.svelte` ŌĆö Current graph implementation (full file, 512 lines, all force layout, node rendering, zoom/pan)
- `02-worktrees/webapp-ui/frontend/src/lib/stores/character.svelte.ts` ŌĆö characterState with canonicalCharacters, selectedCharacter, selectCharacter
- `02-worktrees/webapp-ui/frontend/src/lib/types/index.ts` ŌĆö CanonicalCharacterSummary, StoryOverviewResponse
</read_first>

<action>
Modify `02-worktrees/webapp-ui/frontend/src/lib/components/DashboardGraph.svelte` to support linked character rendering:

1. **Imports:** Add imports for `characterState` from `'$lib/stores/character.svelte'` and `CanonicalCharacterSummary` from types.

2. **Graph node types:** Extend the `GraphNode` interface to include optional fields:
   - `canonical_id?: string` ŌĆö set for linked characters
   - `story_count?: number` ŌĆö number of stories this character appears in
   - `color?: string` ŌĆö assigned color for the node

3. **buildGraph function changes:**
   - After building the characterŌåÆstory mapping from the overview response, cross-reference with `characterState.canonicalCharacters`
   - For characters that have a matching canonical_id: create a SINGLE node (not per-story) using the canonical name, set `canonical_id` and `story_count`
   - For characters without canonical_id: keep current behavior (per-story character nodes with `char:Name` id)
   - Linked character node ids use `canonical:uuid` format to distinguish from `char:Name`
   - Linked character nodes get links to ALL stories they appear in
   - Increase force parameters for linked characters: `forceCollide` radius of 70 for linked chars (vs 60 default), `forceLink` distance of 80 for linked charŌåÆstory edges

4. **Node rendering changes:**
   - Linked characters (`canonical_id` set): render as larger circles (r=28 instead of r=20) with a double-ring effect (outer circle stroke, no fill, r=34), show first letter of canonical name
   - Show story count badge: small circle (r=10) positioned top-right of the node with the count number inside
   - On hover: show full canonical name (same as current hover behavior but with `({story_count} stories)` suffix)
   - Click linked character: call `characterState.selectCharacter(canonical_id)` instead of no-op

5. **Color assignment:**
   - Linked characters use the same PALETTE array but get colors assigned first (before unlinked characters)
   - This gives linked characters visually distinct, predictable colors

6. **Data loading:**
   - Call `characterState.loadCharacters()` in `loadGraph()` alongside the overview API call
   - Use `$effect` to rebuild the graph when `characterState.canonicalCharacters` changes

7. **`handleNodeClick` update:**
   - If node has `canonical_id`: call `characterState.selectCharacter(node.canonical_id)`
   - If node type is 'story': existing behavior (setActiveStory)
   - If node type is 'character' (unlinked): no navigation (existing behavior)
</action>

- Linked character nodes have r=28 with outer ring r=34 (double-ring effect)

- Click on linked character calls `characterState.selectCharacter(canonical_id)`
- Hover on linked character shows canonical name with story count
- Unlinked characters render unchanged (r=20 circles, char:Name ids)
