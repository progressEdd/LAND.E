# Plan 02-05: Enhanced Overview — Backend Linked Character Data and Graph Clustering

## Objective

Enhance the `stories_overview()` endpoint to include canonical character linkage data, enabling the graph to cluster stories by shared characters. Ensure the overview endpoint returns both raw characters (for unlinked) and canonical characters (for linked), so the graph can render both correctly from a single data source.

## Tasks

### Task 1: Enhance stories_overview endpoint with canonical data

</read_first>

<action>
Modify the `stories_overview()` endpoint in `02-worktrees/webapp-ui/backend/app/routers/stories.py`:

1. **Add canonical character data to the response** — After the existing character→story mapping, also query canonical characters:

```python
# Fetch canonical characters with their linked story appearances
canonical_rows = await db.execute_fetchall(
    """SELECT cc.id, cc.canonical_name, csa.story_id
       FROM canonical_characters cc
       JOIN character_story_appearances csa ON cc.id = csa.canonical_id
       ORDER BY cc.canonical_name"""
)

# Build canonical character→story mapping
canonical_to_stories: dict[str, dict] = {}  # id -> {name, story_ids}
for cr in canonical_rows:
    cid = cr["id"]
    if cid not in canonical_to_stories:
        canonical_to_stories[cid] = {
            "id": cid,
            "canonical_name": cr["canonical_name"],
            "story_ids": []
        }
    canonical_to_stories[cid]["story_ids"].append(cr["story_id"])
```

2. **Filter raw characters** — Remove characters from the raw `char_to_stories` mapping that have a confirmed alias (status='confirmed' or 'suggested') in `character_aliases`. These are now represented by their canonical character instead of raw names:

```python
# Get set of (raw_name, story_id) pairs that are aliased
aliased_rows = await db.execute_fetchall(
    "SELECT raw_name, story_id FROM character_aliases WHERE status IN ('confirmed', 'suggested')"
)
aliased_pairs = {(r["raw_name"], r["story_id"]) for r in aliased_rows}

# Filter raw characters: only include characters not fully aliased
filtered_char_to_stories = {}
for name, sids in char_to_stories.items():
    unaliased_sids = [sid for sid in sids if (name, sid) not in aliased_pairs]
    if unaliased_sids:
        filtered_char_to_stories[name] = unaliased_sids
```

3. **Add canonical characters to response** — Add a new field to `StoryOverviewResponse`:

Add to `schemas.py`:
```python
class StoryOverviewCanonicalCharacter(BaseModel):
    """A canonical (linked) character with cross-story presence."""
    id: str
    canonical_name: str
    story_ids: list[str] = Field(default_factory=list)
```

Add to `StoryOverviewResponse`:
```python
canonical_characters: list[StoryOverviewCanonicalCharacter] = Field(default_factory=list)
```

4. **Build and return enhanced response:**
```python
canonical_characters = [
    StoryOverviewCanonicalCharacter(
        id=data["id"],
        canonical_name=data["canonical_name"],
        story_ids=data["story_ids"]
    )
    for data in canonical_to_stories.values()
]

# Use filtered_char_to_stories instead of char_to_stories for raw characters
```

5. **Import the new model** — Add `StoryOverviewCanonicalCharacter` to the imports from schemas.
</action>
