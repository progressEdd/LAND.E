### Task 2: Handle edge cases in character operations

<read_first>
- `02-worktrees/webapp-ui/backend/app/routers/characters.py` — All character endpoints
- `02-worktrees/webapp-ui/frontend/src/lib/stores/character.svelte.ts` — characterState
</read_first>

<action>
Modify `02-worktrees/webapp-ui/backend/app/routers/characters.py`:

1. **DELETE /api/characters/{canonical_id} cleanup:**
   - After deleting a canonical character, also delete any `character_story_appearances` with matching canonical_id (the CASCADE should handle this, but verify)
   - Return the list of affected raw_names so the frontend can refresh the graph

2. **POST /api/characters/link validation:**
   - Check that none of the mentions are already aliased with status='confirmed' to a DIFFERENT canonical character. If so, return 400 with details of the conflict
   - If a mention is already 'suggested' for another canonical, update it (move the alias) rather than creating a duplicate

3. **POST /api/characters/{canonical_id}/split edge cases:**
   - If all mentions are being split away from a canonical, delete the original canonical (it has no aliases left)
   - Validate that mentions_to_split actually belong to this canonical_id (400 if not)

4. **GET /api/characters/candidates robustness:**
   - Handle case where character_mentions has names that have been deleted from stories (orphaned references) — filter these out
   - Handle case where a story has no nodes (freshly created) — skip its character_mentions

5. **Add a POST /api/characters/{canonical_id}/unlink endpoint:**
   - Accept `{ raw_name: string, story_id: string }` body
   - Remove the alias record for this raw_name+story_id combination
   - Remove the character_story_appearances record if canonical_id+story_id match
   - If canonical character has no aliases left, delete it
   - Return updated CanonicalCharacterResponse

6. **Add unlinkCharacter method to rest.ts:**
   ```typescript
   async unlinkCharacter(canonicalId: string, rawName: string, storyId: string): Promise<CanonicalCharacter> {
       return this.request<CanonicalCharacter>(`/api/characters/${canonicalId}/unlink`, {
           method: 'POST',
           body: JSON.stringify({ raw_name: rawName, story_id: storyId })
       });
   }
   ```
   Also add `unlinkMention(canonicalId, rawName, storyId)` method to the `CharacterState` store in `character.svelte.ts` that calls this API method then reloads candidates and characters.
</action>

<acceptance_criteria>
- `/link` returns 400 if a mention is already confirmed to a different canonical
- `/link` updates existing 'suggested' aliases rather than creating duplicates
- `/{canonical_id}/split` deletes original canonical if all mentions split away
- `/{canonical_id}/split` validates mentions belong to this canonical (400 if not)
- `/candidates` filters out orphaned character mentions
- New endpoint `POST /{canonical_id}/unlink` exists
- `/unlink` removes alias and appearance records, deletes canonical if empty
- `rest.ts` has `unlinkCharacter(canonicalId, rawName, storyId)` method calling POST `/api/characters/{canonicalId}/unlink`
- `character.svelte.ts` has `unlinkMention(canonicalId, rawName, storyId)` method that calls the API and refreshes data
- All error responses use HTTPException with appropriate status codes (400, 404)
</acceptance_criteria>
