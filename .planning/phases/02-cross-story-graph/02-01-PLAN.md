   - `id TEXT PRIMARY KEY` â€” UUID
   - `canonical_name TEXT NOT NULL` â€” User-confirmed canonical name (e.g. "Chloe Miller")
   - `description TEXT` â€” Free-text character description (user-editable character bible entry)

   - `id TEXT PRIMARY KEY` â€” UUID
   - `canonical_id TEXT NOT NULL REFERENCES canonical_characters(id) ON DELETE CASCADE`
   - `story_id TEXT NOT NULL REFERENCES stories(id) ON DELETE CASCADE`

   - `role_in_story TEXT` â€” Role context for this character in this specific story
   - `created_at TEXT NOT NULL DEFAULT (datetime('now'))`
   - UNIQUE constraint on `(canonical_id, story_id, raw_name)`

3. `character_story_appearances` table:
   - `id TEXT PRIMARY KEY` â€” UUID
   - `canonical_id TEXT NOT NULL REFERENCES canonical_characters(id) ON DELETE CASCADE`
   - `story_id TEXT NOT NULL REFERENCES stories(id) ON DELETE CASCADE`
   - `role TEXT` â€” Role in this story
   - `context TEXT` â€” How the character appears in this story
   - `arc_notes TEXT` â€” Character arc notes specific to this story
   - `created_at TEXT NOT NULL DEFAULT (datetime('now'))`
   - UNIQUE constraint on `(canonical_id, story_id)`

4. Indexes:
   - `CREATE INDEX IF NOT EXISTS idx_char_aliases_canonical ON character_aliases(canonical_id);`
   - `CREATE INDEX IF NOT EXISTS idx_char_aliases_story ON character_aliases(story_id);`
   - `CREATE INDEX IF NOT EXISTS idx_char_aliases_normalized ON character_aliases(normalized_name);`
   - `CREATE INDEX IF NOT EXISTS idx_char_aliases_status ON character_aliases(status);`
   - `CREATE INDEX IF NOT EXISTS idx_char_appearances_canonical ON character_story_appearances(canonical_id);`
   - `CREATE INDEX IF NOT EXISTS idx_char_appearances_story ON character_story_appearances(story_id);`

Add SQL comments explaining each table's purpose, following the style of 002_graph_support.sql.
