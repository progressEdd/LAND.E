<objective>
Integrate the Tiptap rich text editor with a custom provenance mark for color-coded source tracking.

Purpose: Build the core editor experience — a Tiptap-based rich text editor with markdown formatting (bold, italic, headings) and a custom "provenance" Mark extension that color-codes text by source (AI=white, user=blue, edits=pink, prompt=cream). This is the centerpiece of the NovelAI-style editor experience. The editor replaces the placeholder in +page.svelte.

Output: A working rich text editor in the main pane with formatting toolbar, provenance color-coding, and editor state management.
</objective>

       ```
       cd 02-worktrees/webapp-ui/frontend && npm install @tiptap/core @tiptap/pm @tiptap/starter-kit @tiptap/extension-text-style
       ```

    2. Create `02-worktrees/webapp-ui/frontend/src/lib/extensions/provenance.ts`:
       - Define `ProvenanceSource` type: `'ai_generated' | 'user_written' | 'user_edited' | 'initial_prompt'`
       - Define `PROVENANCE_COLORS` mapping (per user decision):
         - `ai_generated`: `'rgba(255, 255, 255, 0.9)'` (white)
         - `user_written`: `'rgba(100, 149, 237, 0.9)'` (blue — cornflower blue)
         - `user_edited`: `'rgba(255, 182, 193, 0.9)'` (pink — light pink)
         - `initial_prompt`: `'rgba(255, 253, 208, 0.9)'` (cream)
       - Create the `Provenance` Mark using Tiptap's `Mark.create()`:
         - `name: 'provenance'`
         - `addAttributes()`: single attribute `source` with default `'user_written'`
           - `parseHTML`: reads `data-source` attribute from the element
           - `renderHTML`: outputs `data-source` attribute and inline `style` with the mapped color
         - `parseHTML()`: matches `span[data-source]`
         - `renderHTML()`: renders as `<span data-source="..." style="color: ...">` with `0` for content hole
       - Follow the exact pattern from RESEARCH.md Pattern 2, but ensure TypeScript types are correct

    IMPORTANT: The provenance mark is the foundation for color-coded text throughout the app. It must integrate cleanly with Tiptap's undo/redo, copy/paste, and serialization systems. Using the Mark API (not direct DOM manipulation) ensures this.
  </action>

       - Use Svelte 5 runes:
         ```ts
         import type { Editor } from '@tiptap/core';
         
         class EditorState {
           editor = $state<Editor | null>(null);
           isBold = $derived(this.editor?.isActive('bold') ?? false);
           isItalic = $derived(this.editor?.isActive('italic') ?? false);
           // Add derived states for other formatting options
         }
         export const editorState = new EditorState();
         ```
       - Note: The `$derived` reactivity on editor state requires the editor to trigger re-renders via `onTransaction`. This is handled in the Editor component.

    2. Create `02-worktrees/webapp-ui/frontend/src/lib/components/Editor.svelte`:
       - Svelte 5 component with `onMount` / `onDestroy` lifecycle
       - Initialize Tiptap Editor with extensions:
         - `StarterKit` (provides bold, italic, heading, lists, code, blockquote, etc.)
         - `Provenance` (custom mark from step 1)
       - Bind to a DOM element via `bind:this`
       - On `onTransaction`, update `editorState.editor` to trigger reactive updates in toolbar
       - On `onDestroy`, call `editor.destroy()`
       - Style the editor element to fill its container: `min-h-full`, prose-like padding, dark background (bg-gray-950), text color based on provenance (default text-gray-200)
       - Apply Tiptap's `.ProseMirror` CSS: remove outline on focus, set caret-color for dark mode, set min-height
       - Accept optional `content` prop for initial HTML content
       - Accept optional `onUpdate` callback prop for content change notifications

    3. Create `02-worktrees/webapp-ui/frontend/src/lib/components/EditorToolbar.svelte`:
