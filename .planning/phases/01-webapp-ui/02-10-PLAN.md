<objective>
Replace the GraphPlaceholder component with an interactive node graph visualizer. The graph renders the story's node tree as a connected SVG diagram, highlights the active path, shows character mentions as colored badges, and supports click-to-switch branch navigation.

The backend foundation (02-09) provides:
- `GET /api/stories/{id}/tree` → recursive `TreeResponse` with character summaries
- `PATCH /api/stories/{id}/active-path` → switches to a different branch
- `StoryNode` now includes `character_mentions` and `analysis` fields

This plan builds the frontend visualization layer on top of that API.

**Library choice: `d3-hierarchy` (standalone, ~10KB)**
- Uses `d3.tree()` for layout math only (x/y positioning of nodes in a tree)
- All rendering is Svelte SVG templates — no D3 DOM manipulation
- Keeps bundle small, avoids framework mismatch, leverages Svelte reactivity for updates
</objective>

<task id="2" title="Create NodeGraph.svelte component with tree layout">
<action>
1. Create `frontend/src/lib/components/NodeGraph.svelte`
2. Script section:
   - Import `graphState` and `storyState`
   - Import `hierarchy, tree` from `d3-hierarchy`
   - Use `$effect()` to watch `storyState.activeStoryId` — when it changes, call `graphState.loadTree(id)`
   - Use `$derived` to compute the d3 layout:
     - Create `d3.hierarchy(treeData.root)` from the TreeResponse
     - Apply `d3.tree().nodeSize([nodeWidth, nodeHeight])` to compute x/y positions
     - Output: array of positioned nodes and links for SVG rendering
   - Track `containerWidth` and `containerHeight` via a resize observer (bind:clientWidth/clientHeight on wrapper div)
   - Compute SVG viewBox to center the tree and allow panning
3. SVG rendering:
   - `<svg>` fills the container div, viewBox computed from tree bounds with padding
   - Edges: `<path>` elements with cubic bezier curves connecting parent→child nodes
   - Nodes: `<g>` groups positioned at computed x/y, containing:
     - A rounded rectangle (or circle) as the node shape
     - A short text label (truncated content preview, ~20 chars)
     - Node position number
   - Active path styling: nodes on `treeData.active_path` get bright fill (indigo/blue), others get dim fill (gray)
   - Draft node: dashed border, pulsing animation if currently generating
4. Interaction:
   - `onclick` handler on each node group: if node is NOT on active path, call `graphState.switchBranch()`
   - `cursor: pointer` on clickable (non-active-path) nodes
   - Hover tooltip showing node content preview
5. Empty state: if no active story or tree has no root, show a subtle "Select a story" message
6. Loading state: show spinner/skeleton while tree is loading
</action>
