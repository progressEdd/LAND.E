<objective>
Graph polish and seed persistence — five improvements to the node graph:

![Before — nodes too small, seed colors invisible in light mode, seeds only on last active node](../../../00-supporting-files/images/02-CONTEXT/20260214021104.png)

1. **Increase node sizes** — All nodes (paragraph, character, seed) are too small to read. Increase radii and font sizes ~1.5x with proportional spacing adjustments.

2. **Theme-aware seed colors** — Amber (#fbbf24) dashed lines/circles have poor contrast against white backgrounds. Add `--seed-stroke` CSS custom properties: amber in dark mode, dark amber (#b45309) in light mode.

3. **Persist seeds per-node** — Seeds currently only come from `generationState.lastAnalysis` (the most recent generation). Read seeds from each tree node's `analysis.next_paragraph_seeds` instead, so navigating back to earlier nodes still shows their seeds.

4. **Fix SVG seed layering** — Seed edge lines render over seed node circles because they share the same `{#each}` block. Split into two passes: edges first, then circles on top.

5. **Resizable panes** — Verify and improve the Splitpanes splitter visibility with theme-aware colors.
</objective>
