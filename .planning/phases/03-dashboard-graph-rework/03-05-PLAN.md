# Plan 05: Character→Paragraph Edges

## Objective

Extend the NodeGraph visualizer to draw color-coded straight-line edges from character supernodes to the paragraph nodes where they appear. Edges appear on hover over a character supernode (D-05, D-06, D-07) and coexist with tree edges using different styling.

## Tasks

### Task 1: Add hover state and edge computation

<read_first>
- 02-worktrees/webapp-ui/frontend/src/lib/components/NodeGraph.svelte
- 02-worktrees/webapp-ui/frontend/src/lib/stores/graph.svelte.ts
</read_first>

<action>
Add character hover tracking and computed character edges to the NodeGraph component.

1. Add hover state after the existing tooltip state (~line 170):
```typescript
let hoveredCharacter = $state<string | null>(null);
```

2. Add a derived for character edges. Place this after the `layout` derived block (~line 165, before the `truncate` helper function):

```typescript
	interface CharEdge {
		x1: number; y1: number;
		x2: number; y2: number;
		color: string;
	}

	const charEdges = $derived.by((): CharEdge[] => {
		const l = layout;
		if (!l || !hoveredCharacter) return [];

		const char = l.characters.find((c) => c.name === hoveredCharacter);
		if (!char) return [];

		const paraMap = new Map(l.paragraphs.map((p) => [p.id, p]));
		const edges: CharEdge[] = [];

		for (const nodeId of char.nodeIds) {
			const para = paraMap.get(nodeId);
			if (para) {
				edges.push({
					x1: char.x,
					y1: char.y,
					x2: para.x,
					y2: para.y,
					color: char.color,
				});
			}
		}

		return edges;
	});
```

3. Modify the character supernode `<g>` element to set/unset `hoveredCharacter` on mouse enter/leave. Currently the character group has:
```svelte
onmouseenter={(e) => showCharTooltip(e, c)}
onmouseleave={hideTooltip}
```

Change to:
```svelte
onmouseenter={(e) => { hoveredCharacter = c.name; showCharTooltip(e, c); }}
onmouseleave={() => { hoveredCharacter = null; hideTooltip(); }}
```

This tracks which character is hovered AND shows the tooltip simultaneously.
</action>

<acceptance_criteria>
- `NodeGraph.svelte` has `let hoveredCharacter = $state<string | null>(null)`
- `NodeGraph.svelte` has a `CharEdge` interface with `x1`, `y1`, `x2`, `y2`, `color`
- `NodeGraph.svelte` has a `charEdges` derived that returns `CharEdge[]`
- `charEdges` returns empty array when `hoveredCharacter` is null
- `charEdges` returns edges only for the hovered character
- Each edge connects character `(x, y)` to paragraph `(x, y)` using `nodeIds` lookup
- Character supernode `onmouseenter` sets `hoveredCharacter = c.name`
- Character supernode `onmouseleave` sets `hoveredCharacter = null`
- Tooltip still works (showCharTooltip/hideTooltip calls preserved)
</acceptance_criteria>

### Task 2: Render character edges in SVG

<read_first>
- 02-worktrees/webapp-ui/frontend/src/lib/components/NodeGraph.svelte
</read_first>

<action>
Add a new SVG layer for character edges. This layer goes BETWEEN the tree edges (Layer 1) and seed edges (Layer 2), so character edges are behind nodes but above tree edges.

Find the SVG `<g>` with transform. The current layer order is:
1. Tree edges
2. Seed edges
3. Paragraph nodes
4. Character supernodes
5. Seed nodes

Insert the character edges layer AFTER tree edges and BEFORE seed edges. Add this block between the tree edges `{/each}` and the seed edges comment:

```svelte
<!-- Layer 1.5: Character→paragraph edges (hover-only) -->
{#each charEdges as ce, i (`char-edge-${hoveredCharacter}-${i}`)}
	<line
		x1={ce.x1} y1={ce.y1}
		x2={ce.x2} y2={ce.y2}
		class="char-edge"
		style="stroke: {ce.color}"
	/>
{/each}
```

Add the `.char-edge` style in the `<style>` block, after the `.tree-edge-active` style:

```css
/* ---- Character→paragraph edges (hover-only) ---- */
.char-edge {
	stroke-width: 1.5;
	opacity: 0.4;
	pointer-events: none;
	stroke-dasharray: 6 3;
}
```

This styling distinguishes character edges from tree edges (which are solid lines with lower opacity). The dashed pattern and lower opacity make them visually secondary. The color is set inline from the character's assigned color via the `style` attribute.
</action>

<acceptance_criteria>
- Character edges SVG block exists between tree edges and seed edges in the `<g>` transform group
- Each edge is a `<line>` element with class `char-edge`
- Edge stroke color is set inline from `ce.color`
- Edge `stroke-width` is `1.5`
- Edge `opacity` is `0.4`
- Edge `stroke-dasharray` is `6 3` (dashed)
- Edge `pointer-events` is `none` (edges don't intercept mouse events)
- Edges are only rendered when `hoveredCharacter` is not null (charEdges is empty otherwise)
- Each edge has a unique key: `char-edge-${hoveredCharacter}-${i}`
- `.char-edge` CSS class is defined in the `<style>` block
</acceptance_criteria>

## Verification

1. Start the app and open a story that has character mentions (a story with at least 3-4 paragraphs and characters)
2. Look at the right side of the graph — character supernodes should be visible with their colored circles
3. Hover over a character supernode:
   - Edges should appear as dashed lines from the character to each paragraph where they're mentioned
   - Edges should be the same color as the character supernode
   - Tooltip should show "Appears in N paragraph(s)"
   - Character circle should become slightly brighter
4. Move mouse away from the character:
   - Edges should disappear immediately
   - Character circle returns to default appearance
5. Hover over a different character — different color edges should appear
6. Verify edges don't interfere with clicking paragraph nodes or seed nodes
7. Test with dark and light themes — edges should be visible in both
8. Test with a story that has NO character mentions — no edges should ever appear
9. Test zoom/pan — edges should transform with the rest of the graph

## must_haves

- Character→paragraph edges render as straight color-coded lines (D-05)
- Edges appear only on hover over character supernode (D-06)
- Character edges and tree edges coexist with different visual styling (D-07: dashed vs solid)
- Edges are properly positioned using existing layout coordinates
- No performance degradation (edges computed reactively, only for hovered character)
- Works in both dark and light themes
