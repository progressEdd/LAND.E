# Plan 04: Navigation + SettingsPanel Refactor

## Objective

Add navigation elements to return from editor to dashboard (home icon in toolbar, breadcrumb in top bar). Refactor SettingsPanel to remove the story management section since the dashboard now handles story CRUD.

## Tasks

### Task 1: Add home icon to EditorToolbar

- 02-worktrees/webapp-ui/frontend/src/lib/stores/story.svelte.ts
</read_first>

<action>
Add navigation to the EditorToolbar so users can return to the dashboard.

1. Import `storyState`:
```typescript
import { storyState } from '$lib/stores/story.svelte';
```

2. Add a home button as the FIRST element in the toolbar, before the format buttons. Add a separator between the home button and format buttons:

In the template, at the beginning of `<div class="toolbar">`, add:
```svelte
<button
    class="toolbar-btn home-btn"
    onclick={() => storyState.clearActiveStory()}
    title="Back to Dashboard (Home)"
>
    &#x2302;
</button>
<div class="separator"></div>
```

The `&#x2302;` character is ⌂ (house outline). It's a clean, minimal home icon that works at small sizes.

3. Add the `.home-btn` style:
```css
.home-btn {
    font-size: 16px;
    color: var(--text-muted, #9ca3af);
}

.home-btn:hover {
    color: #a5b4fc;
}
```

4. Add a navigation object to the toolbar for the active story title (needed for D-09 breadcrumb). The toolbar already has access to `storyState` — add a derived for the active story title:
```typescript
let activeStoryTitle = $derived(storyState.activeStory?.title ?? '');
```

Display the title in the toolbar after the home button separator, before the format buttons:
```svelte
{#if activeStoryTitle}
    <span class="story-title-breadcrumb">{activeStoryTitle}</span>
    <div class="separator"></div>
{/if}
```

5. Add the breadcrumb style:
```css
.story-title-breadcrumb {
    font-size: 13px;
    font-weight: 600;
    color: var(--text-primary, #e5e7eb);
    white-space: nowrap;
    overflow: hidden;
    text-overflow: ellipsis;
    max-width: 200px;
}
```
</action>
